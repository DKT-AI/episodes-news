### 1. Anthropic hands Claude Code more control, but keeps it on a leash | TechCrunch
Anthropic представил новый режим **auto mode** для Claude Code, который использует ИИ для автоматической проверки действий перед их выполнением, предотвращая рискованные операции и атаки вроде внедрения команд. Режим доступен для тестирования и рекомендуется использовать в изолированных средах.
**Ссылка:** https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/

### 2. Announcing Ingress2Gateway 1.0: Your Path to Gateway API | Kubernetes
Объявлено о выходе версии **1.0 Ingress2Gateway**, инструмента для миграции из Ingress-NGINX в **Gateway API**. Инструмент поддерживает более 30 аннотаций Ingress-NGINX, обеспечивает интеграционные тесты на поведенческом уровне и предоставляет предупреждения о непереводимых конфигурациях. Пользователям рекомендуется тщательно проверять сгенерированные манифесты и постепенно переносить трафик перед полным отказом от Ingress-NGINX.
**Ссылка:** https://kubernetes.io/blog/2026/03/20/ingress2gateway-1-0-release/
**Тезисы для обсуждения:**
- Каковы основные преимущества перехода на Gateway API по сравнению с Ingress-NGINX?
- Какие вызовы могут возникнуть при миграции существующих конфигураций Ingress?
- Насколько полезны предупреждения о непереводимых конфигурациях для плавного перехода?

### 3. Trivy vulnerability scanner breach pushed infostealer via GitHub Actions
Злоумышленники использовали компрометированные учетные данные для внедрения вредоносного ПО через официальные релизы и GitHub Actions сканера уязвимостей **Trivy**. Это позволило им собрать конфиденциальные данные, включая ключи SSH и облачные учетные данные, и обеспечить постоянный доступ к системам через systemd-сервис.
**Ссылка:** https://www.bleepingcomputer.com/news/security/trivy-vulnerability-scanner-breach-pushed-infostealer-via-github-actions/
**Тезисы для обсуждения:**

- Какие меры безопасности можно предпринять для предотвращения подобных атак через GitHub Actions?
