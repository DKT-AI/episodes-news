### 1. Claude Code... Слили?
**Краткое описание:** В статье рассматривается утечка исходного кода компании Anthropic, которая раскрыла детали продвинутых моделей ИИ, включая модель Капибара с контекстным окном в 1 млн токенов и автономные агенты. ### 3. Announcing Ingress2Gateway 1.0: Your Path to Gateway API | Kubernetes
**Краткое описание:** Объявлено о выходе версии 1.0 Ingress2Gateway, инструмента для миграции из Ingress-NGINX в Gateway API. Инструмент поддерживает более 30 аннотаций Ingress-NGINX, обеспечивает интеграционные тесты на поведенческом уровне и предоставляет предупреждения о непереводимых конфигурациях.
**Ссылка:** https://kubernetes.io/blog/2026/03/20/ingress2gateway-1-0-release/

### 4. Trivy vulnerability scanner breach pushed infostealer via GitHub Actions
**Краткое описание:** Злоумышленники использовали компрометированные учетные данные для внедрения вредоносного ПО через официальные релизы и GitHub Actions сканера уязвимостей Trivy. Это позволило им собрать конфиденциальные данные, включая ключи SSH и облачные учетные данные, и обеспечить постоянный доступ к системам через systemd-сервис.
**Ссылка:** https://www.bleepingcomputer.com/news/security/trivy-vulnerability-scanner-breach-pushed-infostealer-via-github-actions/
**Тезисы для обсуждения:**

- Как компании могут улучшить мониторинг и обнаружение компрометированных инструментов?

### 5. Trivy Compromised a Second Time - Malicious v0.69.4 Release, aquasecurity/setup-trivy, aquasecurity/trivy-action GitHub Actions Compromised - StepSecurity
**Краткое описание:** Обнажено компрометирование GitHub Actions trivy-action и setup-trivy, а также бинарника Trivy v0.69.4 через внедрение кредитного стеллера. Атакованные действия читали память Runner.Worker для извлечения секретов и отправляли их на контролируемый злоумышленником домен.

### 6. Security Update: Suspected Supply Chain Incident | liteLLM
**Краткое описание:** Обнаружено компрометирование пакетов litellm==1.82.7 и litellm==1.82.8 на PyPI, содержащих вредоносный код для кражи секретов. Рекомендуется проверить системы на наличие файлов litellm_init.pth и трафика на models.litellm.cloud, а также заменить версии на безопасные, такие как v1.82.6 или ниже.

### 7. Introducing account regional namespaces for Amazon S3 general purpose buckets | AWS News Blog
**Краткое описание:** AWS представил возможность создания общих хранилищ S3 в региональном пространстве имен аккаунта, упрощая управление именами хранилищ. Пользователи могут предсказуемо создавать имена хранилищ, добавляя уникальный суффикс аккаунта и региона. Это обеспечивает уникальность имен и безопасность с помощью политик IAM и AWS Organizations, поддерживается через AWS CLI, SDK и инструменты инфраструктуры как код, такие как CloudFormation.
**Ссылка:** https://aws.amazon.com/blogs/aws/introducing-account-regional-namespaces-for-amazon-s3-general-purpose-buckets/
**Тезисы для обсуждения:**

### 8. [Security]: CRITICAL: Malicious litellm_init.pth in litellm 1.82.8 — credential stealer · Issue #24512 · BerriAI/litellm
**Краткое описание:** Обнаружен критический компромисс цепочки поставок в пакете PyPI litellm версии 1.82.8, содержащий файл `.pth`, который автоматически выполняет скрипт кражи учетных данных при запуске интерпретатора Python. Злоумышленник собирает конфиденциальные данные и отправляет их на удаленный сервер.
