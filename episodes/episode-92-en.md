### 2. Announcing Ingress2Gateway 1.0: Your Path to Gateway API | Kubernetes
Объявлено о выходе версии **1.0 Ingress2Gateway**, инструмента для миграции из Ingress-NGINX в **Gateway API**. Инструмент поддерживает более 30 аннотаций Ingress-NGINX, обеспечивает интеграционные тесты на поведенческом уровне и предоставляет предупреждения о непереводимых конфигурациях. Пользователям рекомендуется тщательно проверять сгенерированные манифесты и постепенно переносить трафик перед полным отказом от Ingress-NGINX.
**Link:** https://kubernetes.io/blog/2026/03/20/ingress2gateway-1-0-release/
**Talking Points:**
- Какой функционал Ingress-NGINX поддерживает инструмент Ingress2Gateway и какие аннотации пока не поддерживаются?
- Как проводятся интеграционные тесты на поведенческом уровне и насколько они надежны для выявления проблем миграции?
- Какие лучшие практики рекомендуется применять при миграции, чтобы минимизировать риски для производительности и доступности приложения?

### 3. Trivy vulnerability scanner breach pushed infostealer via GitHub Actions
Злоумышленники использовали компрометированные учетные данные для внедрения вредоносного ПО через официальные релизы и GitHub Actions сканера уязвимостей **Trivy**. Это позволило им собрать конфиденциальные данные, включая ключи SSH и облачные учетные данные, и обеспечить постоянный доступ к системам через systemd-сервис.
**Link:** https://www.bleepingcomputer.com/news/security/trivy-vulnerability-scanner-breach-pushed-infostealer-via-github-actions/
**Talking Points:**
